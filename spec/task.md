## P0 核心闭环任务

### T001 行业/板块映射体系 ✅ 已完成

**任务类别**：数据基础能力

**目标**：建立“关键词 -> 股票/指数/概念 -> 行业/板块”的映射链路，为行业温度计算提供基础。

**需求拆解**：

- 定义行业、板块、概念、股票、指数的数据结构。
- 建立行业关键词、同义词、别名、简称维护机制。
- 支持股票代码、股票名称、简称、英文名、别名映射。
- 支持概念板块与行业之间的多对多关系。
- 支持产业链上下游标签，例如上游材料、中游制造、下游应用。
- 支持海外标的映射到国内主题，例如英伟达映射 AI、算力、半导体。

**交付物**：

- 行业/板块映射配置或数据表。
- 行业映射查询接口。
- 初始行业样本：AI 科技、新能源、有色金属、半导体、医药、银行、房地产。

**验收标准**：

- 任意舆情内容可根据关键词或实体归属到一个或多个行业/板块。
- 同义词、简称、股票代码均可命中同一主体。
- 支持人工维护和后续扩展。

### T002 温度指数公式 MVP ✅ 已完成

**任务类别**：指标体系

**目标**：实现可解释的行业/板块温度指数，输出 0-100 分。

**需求拆解**：

- 定义温度指数基础公式。
- 将情绪得分、声量异动、传播热度、来源可信度聚合为综合分。
- 初版可暂不接入行情确认，预留行情确认权重。
- 支持按行业、板块、监测方案计算温度。
- 支持按小时、日维度生成温度快照。
- 输出温度分层：冰点、偏冷、中性、偏热、过热。

**建议初版公式**：

```text
板块温度 = 情绪得分 * 35%
        + 声量异动 * 25%
        + 传播热度 * 20%
        + 来源可信度 * 20%
```

**交付物**：

- 温度计算服务。
- 温度快照数据结构。
- 行业/板块温度列表接口。
- 单个行业/板块温度详情接口。

**验收标准**：

- 能对至少 7 个初始行业输出 0-100 温度。
- 温度结果包含各分项贡献。
- 温度分层可解释，不只是单一情绪分数。

### T003 监测方案升级 ✅ 已完成

**任务类别**：产品配置能力

**目标**：将当前关键词监测方案升级为面向行业、板块、概念和股票池的监测方案。

**需求拆解**：

- 监测方案支持选择监测对象：行业、板块、概念、股票池、指数。
- 支持配置核心词、扩展词、排除词。
- 支持配置数据源范围。
- 支持配置数据源权重。
- 支持配置温度阈值和预警阈值。
- 支持配置输出周期：实时、小时级、日级。

**交付物**：

- 监测方案数据结构升级。
- 监测方案创建/编辑表单升级。
- 监测方案与行业映射、温度计算打通。

**验收标准**：

- 用户可创建一个“AI 科技情绪监测”方案，并绑定 AI 相关行业/概念。
- 方案可输出命中内容、情绪分布、温度指数和风险内容。

### T004 行业温度 API ✅ 已完成

**任务类别**：后端接口

**目标**：为前端看板和后续分析提供行业温度数据。
